1. Why Returns Over Prices?

In quantitative finance, we almost never model raw prices. We use returns because they are: (1) normalized â€” comparable across assets with different price levels, (2) more stationary â€” closer to a mean-reverting process, and (3) additive â€” log returns sum across time.

2. Simple (Arithmetic) Returns

R = (P_t - P_{t-1}) / P_{t-1}

Killer feature: Portfolio return = weighted sum of component returns. This makes simple returns essential for cross-sectional analysis (e.g., factor models, portfolio optimization).

3. Logarithmic (Continuously Compounded) Returns

r = ln(P_t / P_{t-1})

Killer feature: Multi-period return = sum of single-period returns. No compounding math needed. Also, log returns are symmetric: a +5% log return followed by -5% brings you back to the start (not true for simple returns!).

ğŸ’¡ The symmetry argument:
Simple returns: $100 Ã— (1+0.5) Ã— (1-0.5) = $75 (you LOST money!)
Log returns: $100 Ã— exp(0.5) Ã— exp(-0.5) = $100 (back to even!)

This is why log returns are preferred for statistical modeling â€” they don't have the asymmetry bias that inflates large losses.

4. Conversion Between the Two

r_log = ln(1 + R_simple)
R_simple = exp(r_log) - 1

5. Volatility: The Most Important Number in Finance

5.1 Historical (Realized) Volatility

Ïƒ = std(r_1, ..., r_n) Ã— âˆš252

The âˆš252 annualization assumes i.i.d. returns (variance scales linearly with time under independence). This assumption is convenient but wrong â€” volatility clusters.

5.2 Volatility Clustering

Mandelbrot (1963) observed that "large changes tend to be followed by large changes." This is the most robust stylized fact of financial markets. It motivates GARCH models and stochastic volatility models.

5.3 Why Quants Obsess Over Volatility

  • Position sizing: Vol targeting â€” risk budget / Ïƒ = position size
  • Risk management: VaR, CVaR all depend on Ïƒ
  • Options: Ïƒ is THE input to Black-Scholes
  • Strategy regime: High vol â†’ trend following works better; Low vol â†’ mean reversion

5.4 Realized vs Implied Volatility

Realized volatility = backward-looking (what happened).
Implied volatility = forward-looking (what the options market expects).
The difference between them is the Volatility Risk Premium (VRP) â€” one of the most persistent sources of alpha in options trading.

def simple_to_log(simple_returns):
    """Convert simple returns to log returns."""
    return np.log(1 + simple_returns)

def log_to_simple(log_returns):
    """Convert log returns to simple returns."""
    return np.exp(log_returns) - 1

def annualize_vol(daily_returns, periods=252):
    """Annualize daily volatility."""
    return daily_returns.std() * np.sqrt(periods)

def rolling_sharpe(returns, window=252, rf=0.04):
    """Rolling annualized Sharpe ratio."""
    excess = returns - rf / 252
    return (excess.rolling(window).mean() * 252) / (returns.rolling(window).std() * np.sqrt(252))
